# Transcriber

Transcriber is a customized software, that uses [Whisper](https://openai.com/research/whisper)
to transcribe sound and video.
Whisper is an automatic speech recognition (ASR) system trained on 680,000 hours of multilingual
and multitask supervised data collected from the web.
The app incorporates speaker diarization to the output generated by Whisper,
by using [WhisperX](https://github.com/m-bain/whisperX).

## Test Pipeline

There is a GitHub workflow defined in ".github/workflows/python-tests.yml" that does the following
- builds and installs transcriber
- runs transcriber on audio test files
- runs unit test
- runs end-to-end tests on the generated output from the execution of the transcriber application

The workflow runs on PR's, pushes to branches with a PR and pushes to main.

## Setting up transcriber locally

To set up transcriber locally follow these steps.
Download the models for WhisperX.
```bash
pip install --no-cache-dir -U "huggingface_hub[cli]"
mkdir -p $HOME/.cache/torch/pyannote
hf auth login --token YOUR_HF_TOKEN
hf download pyannote/speaker-diarization-community-1 --cache-dir ~/.cache/torch/pyannote/
```
Install the linux tool ffmpeg.
```bash
sudo apt install -y ffmpeg
```
Create a local Python environment.
```bash
python3 -m venv .venv
```
Activate the environment.
```bash
source .venv/bin/activate
```
Install the dependencies.
```bash
pip install .
```
Install the aau-whisper package.
```bash
pip install -e .
```
Create the environment variable (replace with your home dir).
```bash
export PYANNOTE_CACHE_DIR="/home/nikko/.cache/torch/pyannote/"
```
## Running transcriber locally
Running with CPU on the base model.
```bash
python3 app.py -o ./output -i shorts.m4a -m base --no-cuda --no-mps --verbose --threads 4
```
Running with GPU on the small model.
```bash
  python3 app.py -o ./output --merge_speakers --output_format all -i shorts.m4a -m small --verbose --threads 4
```
